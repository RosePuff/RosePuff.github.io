树莓派 Ubuntu Server 20.04 LTS

前期准备

硬件

  • Raspberry Pi 4B 树莓派4B（4G）
  • TF disk 32GB
  • TF 读卡器

主机 or 笔记本

笔记本电脑、主机、PC等

软件&环境

  • Windows10
  • 烧制的TF disk软件（Win32disk）
  • Ubuntu Server 20.04 LTS（下载镜像）
  • VS Code
  • 路由器（开启WIFi）

烧录步骤&配置WIFI

  1. TF Disk插入读卡器，插入笔记本
  2. 打开Win32Disk，选择烧录的镜像，点击Write（两分多钟左右）
  3. 修改网络配置打开system-boot找到network-config文件
# This file contains a netplan-compatible configuration which cloud-init
# will apply on first-boot. Please refer to the cloud-init documentation and
# the netplan reference for full details:
#
# https://cloudinit.readthedocs.io/
# https://netplan.io/reference
#
# Some additional examples are commented out below

version: 2
ethernets:
  eth0:
    dhcp4: true
    optional: true
wifis:
  wlan0:
    dhcp4: true
    optional: true
    access-points:
      "Redmi-3304":
        password: "7752045210"
#      myworkwifi:
#        password: "correct battery horse staple"
#      workssid:
#        auth:
#          key-management: eap
#          method: peap
#          identity: "me@example.com"
#          password: "passw0rd"
#          ca-certificate: /etc/my_ca.pem

其中xxxxx是WiFi名字，yyyy是密码，==双引号要保留==

这样就配置好了网络. 第一次通电的时候, 会通过这些信息连接网络

  1. 配置用户名. ubuntu 留有一组默认的用户名, 要求用户第一次登陆的时候改掉它。但是第一次接通电源的时候是无法连接到 WiFi 的, 需要断电再上电. 此时通过 SSH 登陆会遇到问题 passwd: Authentication token manipulation error. 因此, 我们需要使默认的用户名密码, 再坚持一会儿，坚持到下一次登陆。打开文件 user-data, 找到

    chpasswd:
  expire: true
  list:
  - ubuntu:ubuntu

    把 expire: true 改为 expire: false 这样系统可以用这组默认用户名密码 ubuntu:ubuntu 登陆系统, 直到修改了密码

  2. 保存文件，弹出 TF disk。（注意是==弹出==而不是直接拔）

  3. 把 TF disk 插入树莓派 RaspBerry Pi。

  4. 接通电源。

  5. 通过主机的热点界面观察树莓派是否接入。 (这次是不会接入的:P)

  6. 等待 2 分钟, 树莓派上的绿色指示灯不再闪烁，断电

  7. 再次接通电源。

  8. 这样, 树莓派的IP也获得了. SSH 登陆树莓派.

  9. 修改默认密码！这很重要！避免被恶意攻击！

sudo passwd ubuntu

设置远程ROOT登录

查看ssh服务器是否安装并启动

sudo ps -e | grep ssh

安装ssh服务器（一般已经安装好了）

sudo apt-get install openssh-server

安装ssh客户端

sudo apt-get install openssh-client

配置sshd文件

sudo vim /etc/ssh/sshd_config

PermitRootLogin yes （默认为#PermitRootLogin prohibit-password）

更换国内源

sudo cp /etc/apt/sources.list /etc/apt/sources_init.list

进入更改源文件

sudo vim /etc/apt/sources.list

更新

sudo apt-get update
sudo apt-get upgrade

有可能出现的坑

lock锁住了进程

E: Could not get lock /var/cache/

apt/archives/lock - open (11 Resource temporarily unavailable)
E: Unable to lock the download directory

是因为上个操作锁住了apt进程，可以先清理lock

sudo rm -rf /var/cache/apt/archives/lock
sudo apt-get update

正常更新软件

无法install软件

有可能是出现之前的更新的时候没有成功，但是再次更新的时候显示无更新

把备份的重新cp之后采取从新来过的，更换网易的，或阿里的源

配置中文

安装之前先查看语言环境

echo $LANG

中文包

language-pack-zh-hans 简体中文

language-pack-zh-hans-base

language-pack-zh-hant 繁体中文

language-pack-zh-hant-base

安装中文包

sudo apt-get install language-pack-zh-han*

运行语言支持检查

sudo apt install $(check-language-support)

修改配置文件：

sudo vim /etc/default/locale

将原始内容注释掉或删掉，替换为如下内容：

LANG="zh_CN.UTF-8"
LANGUAGE="zh_CN:zh"
LC_NUMERIC="zh_CN"
LC_TIME="zh_CN"
LC_MONETARY="zh_CN"
LC_PAPER="zh_CN"
LC_NAME="zh_CN"
LC_ADDRESS="zh_CN"
LC_TELEPHONE="zh_CN"
LC_MEASUREMENT="zh_CN"
LC_IDENTIFICATION="zh_CN"
LC_ALL="zh_CN.UTF-8"

修改环境文件:

sudo vim /etc/environment

不修改原有内容！！！直接在原内容下面新开一行加入下列内容：

LANG="zh_CN.UTF-8"
LANGUAGE="zh_CN:zh"
LC_NUMERIC="zh_CN"
LC_TIME="zh_CN"
LC_MONETARY="zh_CN"
LC_PAPER="zh_CN"
LC_NAME="zh_CN"
LC_ADDRESS="zh_CN"
LC_TELEPHONE="zh_CN"
LC_MEASUREMENT="zh_CN"
LC_IDENTIFICATION="zh_CN"
LC_ALL="zh_CN.UTF-8"

重启

reboot

配置ROOT登录

  1. 执行sudo passwd修改root账号的密码；
  2. 执行su -切换到root账号；
  3. 编辑文件/etc/ssh/sshd_config，增加下面的
PermitRootLogin yes

安装MongoDB

# 更新并安装
sudo apt-get update
sudo apt install dirmngr gnupg apt-transport-https ca-certificates software-properties-common
sudo apt-get mongodb
# 导入存储库钥匙
wget -qO - https://www.mongodb.org/static/pgp/server-4.4.asc | sudo apt-key add -
sudo add-apt-repository 'deb [arch=amd64] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/4.4 multiverse'

MongoDB的最新版本是4.4版。 要安装另一个版本，请用适当的版本替换4.4。 启用存储库后，通过输入以下命令安装mongodb-org元软件包

sudo apt install mongodb-org

MongoDB在安装的时候会在/etc/mongod.conf创建文件这个就是配置文件了，数据目录是/var/lib/mongodband日志目录/var/log/mongodb

验证是否启动成功，请输入以下命令

mongo --eval 'db.runCommand({ connectionStatus: 1 })'
MongoDB shell version v4.4.0
connecting to: mongodb://127.0.0.1:27017/?compressors=disabled&gssapiServiceName=mongodb
Implicit session: session { "id" : UUID("2af3ab0e-2197-4152-8bd0-e33efffe1464") }
MongoDB server version: 4.4.0
{
  "authInfo" : {
    "authenticatedUsers" : [ ],
    "authenticatedUserRoles" : [ ]
  },
  "ok" : 1
}

ok字段中的值为1表示成功

卸载

sudo service mongod stop
sudo apt-get purge mongodb-org*
sudo rm -r /var/log/mongodb
sudo rm -r /var/lib/mongodb
